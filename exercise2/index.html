Exercise 2: Boxmodel

In the previous exercise we explored some element archetypes. This time we will look more into the block type.

The block type has a certain "box model"

You can see it as layers around the content of the element. From outside to inside:

  1. Margin
  2. Border
  3. Padding
Margin

Content

When you set an element's width, you are setting the width of the content part. So if for instance, you set the width to 50%, and add a border and padding, the outer size of the element will exceed the 50%.

In the earlier days of HTML there was a difference between Internet Explorer and "other browsers" (Netscape, Opera). In Internet Explorer, the width of an element was the width including padding and border, so the padding and border subtracted from the width of the content.

Nowadays all browsers adhere that the defined width is the width of the content, and padding and border widths are added on top of that. (same applies to height by the way)

Margin

Margin is something special. It indicates how much space there should be between an element and another element. But only for elements with "content". And by that I mean something that gives the element a dimension.
