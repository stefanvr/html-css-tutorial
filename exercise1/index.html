Exercise 1: Block / Inline

This exercise is about semantic markup. This file is built up using several different HTML tags. These tags have some default configuration, made up by the HTML Standards organisation and/or browsers.

It is important to know that these properties can be manipulated using CSS. The most important aspect is the way a element is displayed. This is the way an element controls the flow of content within the HTML document.

There are a lot of different tags to explain the semantics of a document, like:

  • Paragraphs
  • Emphasis
  • Lists
  • Tables
  • Images
  • Headers

As you can see, lists break the flow text content, and take the whole width of a page. The next paragraph is shown underneath it. The same applies with paragraphs and headers.

Other elements can be used within the flow of text. Like Emphasis, or an image . A sentence can be continuated, and it does not break the "flow"

In CSS this is called display: block; for elements that take full width (like paragraphs and headers) or display: inline;. For elements that can be used within a sentence.

Next to breaking the flow or keeping it, there are other characteristics with inline or block. For instance. an inline element does not have a width or height. Check the element 'inline' in the source of this document. It has a large width and height, but the browser did not respect them.

The most basic block element with no additional styling is the <div/>. The most basic inline element is the <span/>.

But because these flow properties can be manipulated, you could create a markup with totally different semantics, and change its appearance using

CSS

this way you could use an

Emphasis

tag to display as paragraph, and vise-versa.

There are of-course more variants than inline or block. An image for example is not a pure 'inline' element, since it can have a width and height. this is an inline-block
